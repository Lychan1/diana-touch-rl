# 方法说明与踩坑记录

## 任务演进（V1 → V8，详见 README）

触碰目标体任务是整个项目的第一个 RL 里程碑，第 7 周正式设定任务并跑通 PPO 闭环，最终落地版本为 V8：

- **V1（基线）**：指数型接近奖励 `exp(-25·d)·3`，场景 `rl_env_rl.usd`，桌面高度 0.05；智能体盲目探索，不收敛。
- **V2（可达性）**：抬高目标方块（换场景 `rlenvnew.usd`，桌面高度 0.05→0.10），仍不收敛 → 排除"够不着"。
- **V3（质心碰撞悖论）**：代码以方块*几何中心*算距离，但物理碰撞外壳挡住手臂，使其永远到不了极小成功阈值——定位根因（未单独存版）。
- **V8（最终版）**：换更精细场景 `rlenvnewv8.usd`；成功阈值 0.05→0.12m（指尖基准，含方块半径，完全对齐物理碰撞）；proximity 偏移同步 0.05→0.12；最终采用固定目标位置训练。

**核心经验**：仿真里的成功判定必须对齐底层物理碰撞，否则奖励梯度再好也救不回"物理上到不了"的任务。V8 是这一经验的最终落地——用指尖 + 放宽阈值让"触碰"真正可达。

## 环境与资产

- NVIDIA Isaac Sim 4.5 Standalone，Ubuntu 22.04。
- 机器人：Diana 双臂 + 灵巧手；USD 母版由 URDF 经 Convex Hull 碰撞体转换得到。
- 自碰撞通过 PhysX API 在环境初始化时开启（`PhysxArticulationAPI`）。
- 观测：7 活动关节角(÷π) + 指尖→目标 3D 相对位置 + 欧氏距离 = 11 维。
- 动作：左臂 7 活动关节位置增量（每步 ×0.05），经 `ArticulationAction(joint_indices=...)` 下发。
- 成功阈值：`distance < success_threshold`，**V8 = 0.12m**（更早版本 0.05m）。

## 踩坑

1. **依赖拉取失败**：pip 装 Isaac Sim 4.0 因云端依赖失败崩溃 → 改用官方 Standalone 离线包。
2. **OOM Killer**：大场景编译瞬间占满内存被强杀 → 挂载 16GB Swap 虚拟内存绕过。
3. **URDF 路径不兼容**：`package://` 相对路径报错 → 重写为绝对路径 / 转 USD 资产。
4. **API 报错**：下发关节指令时接口变更 → 用 `ArticulationAction(joint_indices=...)` 显式指定关节。
5. **质心碰撞悖论**：代码算的是方块质心距离，但碰撞体挡住手臂 → 是收敛的真正卡点。

## 复现要点

- 必须设置 `USD_PATH` 环境变量指向场景文件（V8 对应 `rlenvnewv8.usd`；原代码写死 `/home/lychan/...`，已改为可配置）。
- 训练前确认 Swap 已激活、显存充足。
- 评估脚本会加载 `checkpoints/diana_touch.zip`（即 `diana_v8_ppo.zip`），统计触碰成功率与终止原因分布；成功率阈值与 `env.success_threshold`（0.12m）对齐。
